# M5.3 ACP Bridge Reduction Backlog

## Goal

Reduce ACP bridge from an implicit normal-path dependency to an explicit compatibility fallback, while keeping the ACP public control-plane path as the operator-default posture.

## Delivered Scope

- add explicit ACP execution posture helpers so config, runtime, and operator surfaces describe public-first behavior directly
- keep `resolveSessionAdapter()` public-first and allow bridge only on capability errors when `bridge.acpFallbackEnabled=true`
- update ACP unsupported-adapter messaging so failures point to public-path remediation instead of stale bridge-first scaffolding
- expose ACP execution posture and bridge containment language in `swarm doctor`, `swarm status`, and workflow reports
- add unit coverage for public-only success, explicit bridge fallback, and public-only failure guidance
- preserve milestone e2e regression to ensure bridge containment does not break existing compatibility paths

- `vitest run test/unit` -> 52 files, 301 tests passed
- `vitest run test/e2e` -> 18 files, 25 tests passed
- `npm run build` -> clean

## Residual Notes

- ACP bridge is still present as a compatibility path until `M5.4` defines the minimum supported OpenClaw version for removal
- `subagent` still relies on bridge-backed execution because stable public spawn APIs are not available yet

## M5.4 Carryover

- decide the minimum OpenClaw version that makes bridge-free ACP execution supportable
- determine whether ACP bridge can be deleted without weakening operator lifecycle flows
- make the final keep-or-remove decision for `subagent`
