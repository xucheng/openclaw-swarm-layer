# M2.3 Public API Convergence

## Goal

Reduce private bridge coupling over time by isolating version-pinned internals behind replaceable seams and by tracking which top-level public plugin SDK capabilities become available.

## First Slice Delivered

- bridge version mappings are centralized in `src/runtime/bridge-manifest.ts`
- public API capability detection is centralized in `src/runtime/public-api-seams.ts`
- bridge doctor can now report public API availability alongside internal bridge readiness
- doctor now exposes compatibility metadata and replacement candidates for future public API migration
- bridge adapters now expose explicit replacement boundaries for ACP and subagent paths

## Current Private Coupling Hotspots

- ACP control-plane manager export alias
- internal config loader alias
- subagent helper patching against internal bundle symbols
- version-pinned bridge module path mapping

## Current Replaceable Seams

- `resolveInternalModuleSpec()`
- `detectPublicApiAvailability()`
- bridge-backed adapters for ACP and subagent
- replacement boundary constants in bridge-backed adapters

## Next M2.3 Tasks

1. Surface public API availability more clearly in operator output
2. Add migration notes for switching from bridge internals to public exports
3. Reduce direct assumptions about internal alias names where possible
4. Track tested OpenClaw versions and bridge compatibility in one place
